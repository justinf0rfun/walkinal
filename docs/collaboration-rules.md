# Walkinal 协作规则

## 1. 目的

本文档用于约束后续协作方式，降低长上下文、多阶段迭代时的“跑偏”风险。

核心原则：

- 一次只做一个明确目标
- 不顺手扩展
- 不把其他发现的问题混入当前任务

## 2. 执行规则

### 2.1 单目标执行

每一轮只能锁定一个目标。

示例：

- 只做 `HistoryStore` 分页
- 只做 `tab` 重命名
- 只做 `drafts` 持久化修复

不允许在同一轮里混入：

- 模型清理
- 文档改写
- UI 打磨
- 其他顺手修复

除非这些内容直接阻塞当前目标。

### 2.2 开始前先声明范围

每次开始前，先明确说明：

- 本轮只做什么
- 本轮不做什么

格式要求简洁明确。

例如：

- 本轮只做：`HistoryStore` 最近一页 / 分页读取
- 本轮不做：`sentEntries`、`drafts`、其他 UI 改动

### 2.3 阻塞外不扩展

如果过程中发现其他问题：

- 先记录
- 不立刻展开
- 不主动顺手改

只有在满足下面条件时才允许处理：

- 当前目标被它直接阻塞
- 不解决就无法完成本轮目标

### 2.4 长链任务拆阶段

对于长链任务，必须拆成小阶段推进。

例如性能优化：

1. `history` append-only
2. `drafts` 持久化调度器
3. `history` 分页
4. `history` 搜索 / 过滤

每阶段完成后，再进入下一阶段。

### 2.5 完成后只汇报当前目标

每轮完成后，只汇报：

- 当前目标做了什么
- 是否完成
- 是否通过构建 / 验证

不要自动展开到下一堆建议，除非用户明确要求继续。

## 3. 禁止事项

以下行为默认禁止：

- 看到其他可优化点就顺手修改
- 在性能任务里顺手做功能
- 在功能任务里顺手做文档大改
- 在修一个 bug 时顺手重构整片模型

## 4. 推荐工作流

每轮建议按这个流程：

1. 明确本轮目标
2. 明确不做内容
3. 实施当前目标
4. 构建 / 验证
5. 只汇报本轮结果
6. 等待下一轮目标确认

## 5. 适用范围

本规则适用于：

- 功能迭代
- 性能优化
- 结构清理
- 文档更新
- 数据迁移

后续默认按此规则协作，除非用户明确要求一次处理多项内容。
