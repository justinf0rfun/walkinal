# Walkinal 技术方案文档

## 1. 项目概述

### 1.1 基础信息
- **产品名称**：Walkinal（Walkie + Terminal）
- **Fork 来源**：https://github.com/lcoutodemos/clui-cc（MIT 许可证）
- **定位**：终端 AI 编程的悬浮输入外挂（Floating Input Companion）
- **技术基础**：Electron + Monaco Editor + AppleScript

### 1.2 架构原则
**渐进式改造**：在 Clui CC 成熟架构上最小化改造，**保留所有现有惊艳功能**，仅修改核心发送逻辑。

**改造比例**：保留 90%，改造 10%
- ✅ **完全保留**：左侧工具栏（Skill Hub、截图、文件选择）、多标签架构、悬浮窗口 UI、Monaco 编辑器、历史面板、语音输入、设置系统
- 🔧 **核心改造**：消息展示逻辑（从 AI 对话流改为待发送队列）、发送目标（从包装进程改为外部终端桥接）、本地文件存储

## 3. 核心改造点

### 3.1 改造目标对比

| 维度 | Clui CC（原） | Walkinal（改后） |
|------|--------------|------------------|
| **运行模式** | 包装 Cloud Code 进程，内部完成对话 | 独立于进程之外，仅作为输入层 |
| **消息区域** | 展示用户↔AI 的完整对话历史 | 展示**待发送的消息队列**（草稿箱） |
| **发送行为** | 直接发送给内部子进程 | 通过 AppleScript 发送给外部 Warp |
| **历史存储** | 内部存储（内存/SQLite） | 本地文件 `~/Documents/Walkinal/` |

### 3.2 具体改造清单

#### 改造 1：消息区域逻辑（核心）
**现状**：消息区展示对话流（用户提问 + AI 回复）  
**改为**：消息区仅展示**当前标签的待发送队列**

**交互变更**：
- `Enter`：将输入框内容**添加到消息队列**（作为段落展示在上方），不清空窗口
- `Cmd+Enter`：将消息队列中所有段落**拼接后发送到 Warp**，发送成功后清空当前标签队列

**段落类型**：
- 文本段落（手动输入）
- 截图段落（截图工具生成）
- 文件段落（文件选择器或 @file 生成）

#### 改造 2：发送目标（核心）
**移除**：`claude-process.js` 及相关子进程管理逻辑  
**新增**：`terminal-bridge.js` 模块

**发送流程**：

当前标签消息队列 [文本段, 截图段, 文件段]
↓ 智能拼接（带分隔符）
完整提示词文本
↓ AppleScript → Warp
Cloud Code/Codex CLI 接收处理


#### 改造 3：消息队列编辑功能（新增）
- **删除段落**：悬停段落卡片显示删除按钮（🗑️），点击移除该段
- **编辑段落**：悬停段落卡片显示编辑按钮（✏️），点击后：
  - 该段落内容**回填到输入框**
  - 该段落**从队列中移除**
  - 用户修改后按 `Enter` 重新添加到队列

#### 改造 4：历史记录存储（数据层）
**改为**：本地文件系统存储

**存储结构**：
~/Documents/Walkinal/
├── drafts.json          # 当前所有标签的草稿状态
├── history.jsonl        # 发送历史（JSON Lines 格式，便于追加）
└── config.json          # 用户设置（存储路径可配置）


**历史面板**：复用现有 UI，改为从本地文件读取。

### 4.2 交互逻辑

**段落管理**：
- **添加段落**：每按一次 `Enter`，当前输入框内容成为一个**段落卡片**添加到队列上方，输入框清空
- **删除段落**：悬停段落卡片显示删除按钮（🗑️），点击移除该段
- **编辑段落**：悬停段落卡片显示编辑按钮（✏️），点击后内容回填到输入框，段落从队列移除

**发送逻辑**：
- **发送（↵）**：仅将段落添加到队列（不发送到终端），方便组织多段内容
- **发送并执行（⌘↵）**：将当前标签所有段落拼接，发送至 Warp，成功后清空队列

**左侧工具栏行为**：
- **Skill Hub**：打开面板（保留现有功能）
- **截图**：触发系统截图 → 截图内容作为新段落添加到当前队列
- **附件**：打开系统文件选择器 → 选中文件作为文件段落添加到队列

## 5. 技术实现方案

### 5.1 技术选型

- **基础框架**：Electron（基于 Clui CC Fork 改造）
- **状态管理**：复用现有 Store，改为管理段落队列而非会话状态
- **编辑器**：Monaco Editor（保持现有配置）
- **文件索引**：ripgrep（用于 @file 功能，可选 V2 实现）
- **模糊搜索**：Fuse.js（前端轻量级模糊匹配）
- **文件监听**：chokidar（增量更新索引）
- **终端通信**：AppleScript（macOS）→ Warp

### 5.2 文件改动边界

**需修改文件**：
- `src/main/bridge.js`（新增）：AppleScript 桥接 Warp
- `src/renderer/components/MessageQueue.vue`（改造）：改为展示段落队列，支持编辑/删除操作
- `src/renderer/store/drafts.js`（改造）：段落队列状态管理（支持 CRUD）
- `src/main/history.js`（新增）：本地文件读写历史记录

**完全保留文件**：
- `src/renderer/components/LeftToolbar.vue`（左侧三按钮）
- `src/renderer/components/SkillHub.vue`（Skill Hub）
- `src/renderer/components/ScreenshotTool.vue`（截图）
- `src/renderer/components/FilePicker.vue`（文件选择器）
- `src/renderer/components/HistoryPanel.vue`（历史面板 UI）
- `src/main/voice.js`（语音输入）
- `src/main/window.js`（窗口管理）
- UI 样式、主题、动画效果

### 5.3 关键模块设计

**终端桥接模块**：
```javascript
// 使用 AppleScript 与 Warp 通信
// 利用 Bracketed Paste 模式确保多行文本作为整体输入
const sendToTerminal = async (text) => {
  // 写入剪贴板 + 模拟 Cmd+V 粘贴
  // 或直接使用 AppleScript 的 write text
};

interface Message {
  id: string;
  type: 'text' | 'file' | 'screenshot';
  content: string;
  metadata?: {
    filePath?: string;
    fileSize?: number;
    preview?: string;
  };
}

interface Tab {
  id: string;
  name: string;
```

历史记录服务：
使用 Node.js fs 模块进行 JSONL 追加写入
读取时解析为数组，支持分页和搜索
提供收藏、标签、删除等 CRUD 接口



### 5.4 数据流

用户输入 → 按 Enter → 添加到消息队列（本地状态）
                                ↓
                      点击 Cmd+Enter
                                ↓
                      拼接所有段落
                                ↓
                      AppleScript → Warp
                                ↓
                      AI 工具接收处理
                                ↓
                      保存到 history.jsonl

## 6. 数据存储方案

### 6.1 存储路径
默认：~/Documents/Walkinal/
可配置：用户可在设置面板修改存储目录

### 6.2 文件格式
drafts.json（草稿状态）：
JSON
Copy
{
  "activeTabId": "tab-1",
  "tabs": [
    {
      "id": "tab-1",
      "name": "Feature A",
      "messages": [
        {"type": "text", "content": "请帮我..."},
        {"type": "file", "path": "src/utils.ts", "preview": "..."}
      ],
      "draftContent": "当前输入框未发送内容"
    }
  ]
}
history.jsonl（发送历史）：
每行一条 JSON 记录，便于追加和流式读取

{"id": "uuid-1", "timestamp": "2026-04-17T13:30:00Z", "title": "重构 auth 模块", "content": "完整拼接内容...", "parts": 3, "favorite": false}
{"id": "uuid-2", "timestamp": "...", ...}

### 6.3 存储策略
草稿：应用退出时自动保存，重启时恢复
历史：发送成功后立即追加写入，支持用户手动导出/导入
